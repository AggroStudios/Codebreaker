# CLAUDE.md - Core Project Context

## Project Overview
A web-based game that the player gets to play as a hacker in a world where they are trying to break ciphers and own the network. The project is built with Vite, React, Zustand, Material UI, and TypeScript.

## Essential Commands
- `npm run dev`: Start the development server
- `npm run build`: Build the project
- `npm run preview`: Preview the built project
- `npm test`: Run the tests
- `npm run lint`: Run the linter
- `npm run clean`: Clean the project
- `npm update`: Update the project
- `npm upgrade`: Upgrade the project

## Directory Structure
- `src`: Contains the main source code for the application.
- `src/assets`: Contains the assets for the project.
- `src/components`: Contains the components for the project.
- `src/data`: Contains the data for the project.
- `src/includes`: Contains the interfaces for the project.
- `src/lib`: Contains the utility functions for the project.
- `src/pages`: Contains the pages for the project.
- `src/stores`: Contains the stores for the project.
- `src/theme`: Contains the theme for the project.
- `src/utils`: Contains the utility functions for the project.

## Code & Quality Style Guidelines
- **Language**: Use TypeScript for all code, strictly enforced.
- **State Management**: Use Zustand for state management.
- **Styling**: Use Material UI's styled components for styling.
- **Testing**: Use Vitest for testing.
- **Documentation**: Use JSDoc for documentation.
- **Naming**: Use camelCase for all naming conventions.
- **Spacing**: Use 4 spaces for indentation.
- **Quotes**: Use single quotes for all strings.
- **Imports**: Use named imports for all imports.
- **Exports**: Use named exports for all exports.
- **Constants**: Use all uppercase for constants.
- **Variables**: Use camelCase for variables.
- **Functions**: Use camelCase for functions.
- **Classes**: Use PascalCase for classes.
